üìú Guidelines
To ensure consistency across contributions, please follow these guidelines:

Code Quality
Use PEP 8 guidelines for Python code style.

Ensure that your code is well-documented.

Add tests for new features or bug fixes.

No Hardcoded Secrets
This project does not allow any hardcoded API keys, credentials, or secrets. Please use placeholders or environmental variables.

Documentation
If you add new functionality, update the documentation in README.md, API_REFERENCE.md, or other relevant files.

Keep documentation clear and concise, with code examples where appropriate.

Security & Privacy
Follow the best security practices and ensure that no sensitive data is exposed in the code or commits.

Make sure that your changes are compliant with the DATA_PRIVACY_POLICY.md.
